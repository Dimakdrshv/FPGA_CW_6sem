# FPGA Project (Vivado)

## 📌 Описание

Этот репозиторий содержит исходный код проекта для FPGA (Vivado).

Основная идея:
- хранить **только исходники (src)** и конфигурацию проекта
- вся сборка (synthesis, implementation и т.д.) генерируется локально

---

## 🌿 Ветки и workflow

Используется следующая модель работы с Git:

- `main` — стабильная ветка (production / проверенный код) **(в данную ветку не разрешается делать коммиты, исключительно pull request)**
- `working/<CW_MODULE_NAME>` — рабочие ветки для задач **(название файлов капсом без расширения (.v))**
- `bugfix/<CW_MODULE_NAME>` — ветки для исправления багов **(название файлов капсом без расширения (.v))**

Примеры:
- `working/CW_UART_COUNTER`
- `bugfix/CW_DIVIDER`

## 🚀 Начало работы

Клонирование репозитория выполняется **один раз** перед началом работы:

```bash
git clone <URL_РЕПОЗИТОРИЯ>
cd <НАЗВАНИЕ_ПРОЕКТА>
```

После этого вся дальнейшая работа ведётся **внутри этой копии репозитория**.

---

## 🔁 Общий процесс работы

1. Актуализировать `main`
2. Запустить Tcl-скрипт `add_files.tcl` для обновления `fileset`'ов в проекте Vivado
3. Создать рабочую локальную ветку
4. Сделать изменения
5. Запушить ветку
6. Создать Pull Request (PR)
7. Дождаться ревью
8. После ревью совершить merge в `main`

---

## ⚙️ Запуск скрипта
1. Запустить САПР Vivado
2. Внизу открыть планку `tcl console`
3. Прописать `source ./scripts/add_files.tcl`
4. Дождаться успешного завершения выполнения скрипта

## ⚙️ Основные команды

### 🔄 Обновить `main`
```bash
git checkout main
git pull
```

### 🌱 Создать новую ветку
```bash
git checkout -b <BRANCH_NAME>/<CW_MODULE_NAME>
                или
git switch -c <BRANCH_NAME>/<CW_MODULE_NAME>
```

### 💾 Добавить изменения
```bash
git add .
git commit -m "описание изменений (на английском языке)"
```
**Сообщение в кавычках после флага `-m` должно быть кратким и понятным. Пример: `Added state machine` или же `Updated reg <NAME>` и т.п.**

### 🚀 Запушить ветку
```bash
git push -u origin <BRANCH_NAME>/<CW_MODULE_NAME>
```

### 🔀 Создание Pull Request

После push:

1. Перейти в репозиторий
2. Создать PR:
    - from: <BRANCH_NAME>/<CW_MODULE_NAME>
    - to: main
3. Дождаться ревью
4. После approval совершить `merge`
